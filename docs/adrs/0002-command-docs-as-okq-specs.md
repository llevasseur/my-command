# Command docs as okq specs and per-command feature docs

## Status

Accepted.

## Context

MyCommand commands are agent instruction files (`src/commands/*.md`), and the
install wizard is data-driven over that directory. There was no discoverable,
queryable contract describing how a command behaves, how to add one, or how the
wizard includes it — so command behavior, wizard inclusion, and README/CHANGELOG
could drift apart silently.

## Decision

Maintain a `docs/` [OKF](https://github.com/GoogleCloudPlatform/knowledge-catalog/tree/main/okf)
bundle, indexed with [okq](https://github.com/mikevalstar/okq):

- `specs/` — process specs: [Adding a command](../specs/adding-a-command.md) and
  [Install wizard](../specs/install-wizard.md).
- `features/` — one feature doc per command, capturing its flags/parameters and
  behavior.

Invariants:

- Adding a command requires a feature doc and verified wizard inclusion
  (with overwrite).
- Changing a command's flags or parameters requires updating its feature doc in
  the same change.

## Consequences

The suite is queryable (`okq --bundle docs search "<topic>"`,
`okq --bundle docs find --type feature`). One extra doc step is added per command
and per flag change. Directory `index.md` files are machine-generated by
`okq --bundle docs index` — regenerate rather than hand-edit them.
